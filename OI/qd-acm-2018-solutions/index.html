[B] Kawa Exam

Description

给定一张无向图，每个点有一个目标颜色。现在对于每条边求出：

  • 删除这条边之后，给每个点染色，要求单个联通块内颜色相同，且满足目标颜色的点最多，求最多多少个点满足目标颜色。

n,m⩽105n,m\leqslant10^5n,m⩽105，∑n,∑m⩽106\sum n,\sum m\leqslant10^6∑n,∑m⩽106。2s/64MB

Solution

显然删边之后每个连通块取其众数；而且只有割边删掉答案才会变。

所以边双连通分量缩成一个点之后只需要在树上做这个操作，容易发现要求的其实就是子树众数和子树外众数。

可以直接 dsu on tree 解决，复杂度 O(nlog⁡n+m)O(n\log n+m)O(nlogn+m)。附：Tutorial (dsu on tree)

[C] Flippy Sequence

Description

给出两个长为 nnn 的 01 串，你可以（且必须）在两个 01 串中各选择一个区间反转（0→1,1→00\to1,1\to00→1,1→0），问有多少种方案使操作后两个串相同。

n⩽106,∑n⩽107n\leqslant10^6,\sum n\leqslant10^7n⩽106,∑n⩽107。1s/64MB

Solution

先把两个串异或起来（假设结果串是 CCC），相当于选择两个（有顺序）的区间使得这两个区间异或起来得到两个串的异或。

当 CCC 中只含 0 的时候，任意两个相同的区间都可以，方案数是 n(n+1)2\frac{n(n+1)}22n(n+1)​；

当 CCC 中有一段连续的 1 而其他都是 0 的时候，手玩一下就可以知道恰好有 2n−22n-22n−2 种方案（注意两个区间有顺序）；

当 CCC 中恰有两段连续的 1 的时候，方案数是 666。

当 CCC 中有三段或以上连续的 1 的时候，显然方案数是 000。

[D] Magic Multiplication

Description

对于两个正整数 A,BA, BA,B，若 AAA 的各个数位（从高到低，无前导零）为 a1,a2,…,ana_1,a_2,\dots,a_na1​,a2​,…,an​，BBB 的各个数位为 b1,b2,…,bmb_1,b_2,\dots,b_mb1​,b2​,…,bm​，则定义

A⊗B=a1b1+a1b2+a1b3+⋯+a1bm+a2b1+⋯+a2bm+⋯+anbmA\otimes B=a_1b_1+a_1b_2+a_1b_3+\dots+a_1b_m+a_2b_1+\dots+a_2b_m+\dots+a_nb_m A⊗B=a1​b1​+a1​b2​+a1​b3​+⋯+a1​bm​+a2​b1​+⋯+a2​bm​+⋯+an​bm​

但是式子中的 +++ 表示拼接操作，比如

12+34=1234,0+2=02,23⊗45=8101215,20⊗20=400012+34=1234,0+2=02,23\otimes45=8101215,20\otimes20=400012+34=1234,0+2=02,23⊗45=8101215,20⊗20=4000。

给出 n,mn,mn,m 及 C=A⊗BC=A\otimes BC=A⊗B，求 AAA 和 BBB。若有多解输出 AAA 最小的，还有多解则输出 BBB 最小的。

n,m,∣C∣⩽2×105,∑∣C∣⩽2×106n,m,|C|\leqslant2\times10^5,\sum|C|\leqslant2\times10^6n,m,∣C∣⩽2×105,∑∣C∣⩽2×106。1s/64MB

假设我们知道了 a1a_1a1​。那么如果 c1<a1c_1<a_1c1​<a1​，显然 c1c_1c1​ 和 c2c_2c2​ 两位拼起来才是 a1×b1a_1\times b_1a1​×b1​，由此可以求出 b1b_1b1​。

否则 c1≤a1c_1\leq a_1c1​≤a1​，而如果 a1b1a_1b_1a1​b1​ 进了位也不会进到 a1a_1a1​ 那么多，所以 a1b1=c1a_1b_1=c_1a1​b1​=c1​，也可以求出 b1b_1b1​。

按照这种方式继续求 b2…bmb_2\dots b_mb2​…bm​（注意如果某一次开头一位是 000 的话 bib_ibi​ 也一定是 000），然后反过来用 b1b_1b1​ 求 a2,a3…ana_2,a_3\dots a_na2​,a3​…an​，在中间的部分直接用 aibja_ib_jai​bj​ 去跟当前的位置比较即可。

[E] Plants vs. Zombies

Description

直线上有 nnn 个植物，第 iii 棵植物坐标为 iii，浇一次水会长 aia_iai​ 高。

你最开始在 000 点，执行 mmm 次操作。每次操作必须往左或右走一步并给走到的那棵植物（如果有）浇一次水。

最大化 mmm 次操作后最矮的植物的高度。

n⩽105,m⩽1012,∑b⩽106n\leqslant10^5,m\leqslant10^{12},\sum b\leqslant10^6n⩽105,m⩽1012,∑b⩽106。2s/64MB

Solution

先二分一个高度并求出每个位置至少浇水多少次。

首先我们证明，每种结尾在 nnn 或者 n−1n-1n−1 的方案都能换成（步数不变，每棵植物浇水次数也不变）

1→2→1→⋯→2→3→2→⋯→3→4→⋯→n−1→n→n−11\to2\to1\to\dots\to2\to3\to2\to\dots\to3\to4\to\dots\to n-1\to n\to n-1 1→2→1→⋯→2→3→2→⋯→3→4→⋯→n−1→n→n−1

的形式（如果停在 nnn ，那最后就少走一步 n−1→nn-1\to nn−1→n），即“左右横跳”（雾

这个结论是显然的：既然从 111 走到了 nnn 或 n−1n-1n−1 那么把走过的（有向）边拿出来就是一条欧拉路。不停地贪心“若可以往左走就往左走，否则往右走”一定能走成刚才那种形式。

接下来的结论是：任何方案都可以换成（同上）从 nnn 或 n−1n-1n−1 结尾的方案，从而换成上面那种样子。

如果最后停在了 iii 而 i<n−1i<n-1i<n−1 ，那么由于它一定到过 nnn，所以存在两步是 i+2→i+1i+2\to i+1i+2→i+1 和 i+1→ii+1\to ii+1→i 。

把 i+2→i+1i+2\to i+1i+2→i+1 这一步换成 i→i+1i\to i+1i→i+1（同时交换一下走顺序），显然这还可以构成欧拉路，并且终点变成了 i+2i+2i+2。

重复这个步骤可以让我们最后停在 nnn 或者 n−1n-1n−1，且按上面那种走法走。

那么贪心一下就可以了（每次如果左边的没浇水就走到左边，否则走到右边）。

[F] Tournament

Description

有 nnn 个骑士进行 kkk 轮比赛。每轮比赛中每个骑士都要与另一名骑士对决，并且有两个条件：

  • 两名相同骑士只能对决一次。
  • 如果在第 iii 轮中 a 与 b 对决，c 与 d 对决，而在第 jjj 轮中 a 与 c 对决，那么在第 jjj 轮中 b 必须与 d 对决。

问有没有合法方案。如果有，输出字典序最小的方案。

n,k⩽1000,∑n,∑k⩽5000n,k\leqslant1000,\sum n,\sum k\leqslant5000n,k⩽1000,∑n,∑k⩽5000。1s/64MB

Solution

可以构造对决至多 2t−12^t-12t−1 场的方案，其中 2t∣n,2t+1∤n2^t\mid n, 2^{t+1}\not\mid n2t∣n,2t+1​∣n。

构造方式很简单：第 jjj 轮 iii 匹配 i xor ji{\,\rm xor\,}jixorj，其中 0≤i<n,1≤j≤k0\leq i<n,1\leq j\leq k0≤i<n,1≤j≤k。容易验证这是所有 k≤2t−1k\leq2^t-1k≤2t−1 的字典序最小的方案（实际上相当于贪心构造）。

所以我们现在只需要做的事情是：证明确实最多只能对决 2t−12^t-12t−1 场。

首先我们证明：kkk 轮过后，如果两个骑士对决过就连一条边的话，得到的图里每个连通块大小都是 222 的幂。

可以归纳证明：k=0k=0k=0 是显然的（连通块大小都是 20=12^0=120=1）；假设前 k−1k-1k−1 轮之后每个连通块大小都是 222 的幂。

我们注意到，如果我确定了第 kkk 轮某一个人的对手，那么可以用题面中第二条规则确定他所在连通块（前 k−1k-1k−1 轮构成的图中的连通块，下同）所有人的对手；

并且如果是 a 和 b 对决的话，那么所有 a 连通块里的人都会与 b 连通块里的人对决，从而这两个连通块里的人一样多。

那么如果 a 和 b 对决，而它们本来就在一个连通块，之后还会在同一个连通块，于是根据归纳假设，这个连通块的大小还是 222 的幂；

否则，a 和 b 不在一个连通块，但是他们所在的连通块大小相等。这两个连通块会合并成一个，从而变成了原来的两倍，还是 222 的幂。

于是我们证明了结论。而如果对决了 2t2^t2t 场，每个连通块大小必定大于 2t2^t2t，而又是 222 的幂，从而是 2t+12^{t+1}2t+1 的倍数，与 2t+1∤n2^{t+1}\not\mid n2t+1​∣n 不符。

所以最多只能对决 2t−12^t-12t−1 场。

[I] Soldier Game

Description

nnn 名士兵排成一排，第 iii 名士兵的能力值是 aia_iai​。现在要把他们划成很多队伍，每支队伍要么是一个人，要么是连续的两个人。

一个人的队伍能力值就是这个人的能力值，两个人的队伍的能力值是他们两个能力值的和。

最小化队伍能力值的极差。

n⩽105,∑n⩽106n\leqslant10^5,\sum n\leqslant10^6n⩽105,∑n⩽106。2s/64MB

Solution

可能的队伍显然只有 2n−12n-12n−1 种。我们把队伍处理出来按能力值排序，那么只需要双指针扫一遍，同时用数据结构维护双指针之间的这些队伍能不能拼成整个 111 到 nnn 的区间。

由于队伍只可能是一个人或者相邻两个人，我们可以用线段树维护每个区间：
