UOJ55 [WC2014]紫荆花之恋

发表于 2018-03-14 更新于 2022-06-24 分类于 Solutions 阅读次数：

Description

有一棵树，最开始是一棵空树。有nnn个操作，每次操作给出a,c,ra,c,ra,c,r，表示新的节点父亲编号为aaa，其与父亲之前的边权为ccc，其“感受能力”为rrr。

每次操作后询问满足dist(i,j)⩽ri+rjdist(i, j)\leqslant r_i+r_jdist(i,j)⩽ri​+rj​的(i,j)(i,j)(i,j)有多少对，其中distdistdist表示两点间唯一路径上的边权和。

所有aia_iai​与上次的答案last_anslast\_anslast_ans（初始为000）异或后对10910^9109取模。

Solution

首先，我们发现实际上每次只需要统计新加进的结点有多少合法解。那么，我们将dist(i,j)dist(i, j)dist(i,j)写成dist(i,p)+dist(j,p)dist(i,p)+dist(j,p)dist(i,p)+dist(j,p)，其中ppp是iii和jjj的LCA。

那么，

dist(i,j)⩽ri+rj⇔dist(i,p)+dist(j,p)⩽ri+rj⇔ri−dist(i,p)≥dist(j,p)−rj\begin{aligned}dist(i,j)&\leqslant r_i + r_j\\ \Leftrightarrow dist(i,p)+dist(j,p)&\leqslant r_i+r_j\\ \Leftrightarrow r_i-dist(i,p)&\geq dist(j,p)-r_j\end{aligned}dist(i,j)⇔dist(i,p)+dist(j,p)⇔ri​−dist(i,p)​⩽ri​+rj​⩽ri​+rj​≥dist(j,p)−rj​​

所以，我们枚举新加的结点的所有祖先ppp，计算以ppp为LCA的满足条件的点对(i,j)(i,j)(i,j)有多少个即可。

计算时，先查询ppp子树在添加iii之前有多少jjj满足ri−dist(i,p)≥dist(j,p)−rjr_i-dist(i,p)\geq dist(j,p)-r_jri​−dist(i,p)≥dist(j,p)−rj​，再减去其中LCA不是ppp的（即，iii和jjj在ppp的同一棵子树里，这样我们就要查询在这棵子树里有多少jjj满足此条件）。

然而，要高效地维护这个信息，就需要在每个结点上维护一个Treap（记录所有的dist(j,p)−rjdist(j,p)-r_jdist(j,p)−rj​），但这样当树退化为链，单次加入时间复杂度增加为O(nlogn)O(nlogn)O(nlogn)，空间复杂度增加为O(n2)O(n^2)O(n2)。

于是，借鉴替罪羊树的思想，我们在某个结点子树内的点个数大于其父亲子树内点的个数的α∈(0,1)\alpha\in(0,1)α∈(0,1)倍的时候暴力重构其父亲的子树。既然要求重构之后尽量平衡，理所当然地选用点分治。

这样，上面的分析中所有的“父亲”和“祖先”都应理解为点分治树上的祖先（LCA当然也是），但是答案仍是正确的，因为我们仍然不重不漏地枚举了所有可能的点。

代码实现上，还要写一个倍增LCA以查询distdistdist，因为点分治之后的祖先不一定是原树上的祖先，不能直接用depdepdep相减。
