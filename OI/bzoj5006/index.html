BZOJ5006 [THUWC2017] 随机二分图

发表于 2018-04-11 更新于 2022-06-24 分类于 Solutions 阅读次数：

Description

有一个左右各 nnn 个点的二分图。有 mmm 组有概率出现的边，分为三种：

  1. 一组一条边，出现或不出现的概率各有 50%。
  2. 一组两条边，同时出现或同时不出现的概率各有 50%。
  3. 一组两条边，必定会恰好出现一条，概率各有 50%。

组与组之间的概率是独立的。

问完美匹配的期望个数 EEE ，输出 2nE(mod109+7)2^nE \pmod {10^9+7}2nE(mod109+7) （显然是整数）。

Solution

期望是可加的。所以可以枚举每个完美匹配求出现的概率和。复杂度 O(n∗n!)O(n*n!)O(n∗n!) ， 20pts 。

为什么要枚举完美匹配呢？能不能状压 DP 求 fS1,S2f_{S1,S2}fS1,S2​ 表示左右边各剩下 S1,S2S1, S2S1,S2 时完美匹配个数呢？

因为第 2, 3 种边不好统计。假设只有第 1 种边。O(m∑i=0n(ni)2)O(m\sum_{i=0}^n {n \choose i}^2)O(m∑i=0n​(in​)2) （由于好多状态其实访问不到所以复杂度其实是 O( 玄学 ) ），配合上面那个有 40pts 。

考虑有第 2, 3 种边。如果我枚举的匹配中，不存在某两条边在同一组，那么它的概率肯定是 2−n2^{-n}2−n 。

那么我考虑暂时把第 2, 3 种边拆开成两条独立的边，看对答案有什么影响。

如果某个匹配同时用到了第 2 种边某组中的两条（两条边的端点显然互不相同），那么应该有 12\frac1221​ 的概率。但是如果我把它看成独立的两条边的话就是 14\frac1441​ 的概率。所以我新建“一条边”连着这 4 个点，出现概率为 14\frac1441​ 。这样一来这个匹配就会被算两遍，每遍都有 14\frac1441​ ，一共就是 12\frac1221​ 。

同样的，如果某个匹配同时用到了某组第 3 种边的两条，那么它会多算 14\frac1441​ 的概率。那么我新建“一条边”连接这 4 个点，出现概率为 −14-\frac14−41​ 。（似乎很魔幻）这样就可以把多出来的那 14\frac 1441​ 抵消掉了。

然后状压 DP 就好了。

状态开不下？开 map 啊！
