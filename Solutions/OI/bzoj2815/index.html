BZOJ2815 [ZJOI2012] 灾难

发表于 2018-04-13 更新于 2022-06-24 分类于 Solutions 阅读次数：

Description

nnn 个点 mmm 条边的DAG，对于每个点，求它的重要度。重要度定义如下：

对于 vvv 这个点，将其（和其相连的所有边）删去。之后考察所有点，如果某个点初始时出度不为 000 而当前出度为 000 ，则将这个点也删去。重复这个过程直到没有点被删去。 vvv 的重要度即为除它以外删掉的点的个数。

n⩽105n\leqslant10^5n⩽105 。

Solution

首先我们把所有边都反向（这样比较方便处理，其实也无所谓的）。

这样的话，拓扑序后面的点不会影响到前面的点。

首先，我们有一个性质： uuu 会影响到 vvv 当且仅当所有从生产者到 vvv 的路径上都包含 uuu 。这是显然的。

对于一个点 vvv ，找到会影响到它的点（也就是说，那些删掉之后会导致 vvv 消失）中除了它本身外拓扑序最靠后的一个点，假设它是 tvt_vtv​ 。

那么有一个重要的性质：会影响到 vvv 的点只有 v,tv,ttv,...v, t_v, t_{t_v}, ...v,tv​,ttv​​,... 这些。

证明：

如果有某个点 uuu 会影响到 vvv ，它在拓扑序上必定位于某个 tkt_ktk​ 和 kkk 之间，其中 k=t⋱vk=t_{\ddots_v}k=t⋱v​​ 。

既然 tkt_ktk​ 是最后一个能影响到 kkk 的，那么 uuu 就不能影响到 kkk 。那么一定存在一条从生产者到 kkk 的路径不经过 uuu 。但是既然 kkk 会影响到 vvv ，就一定存在一条 kkk 到 vvv 的路径。把这两条路径拼起来，就能得到一条从生产者到 vvv 、不经过 uuu 的路径，于是 uuu 就不能影响到 vvv ，矛盾。

那么我们把 ttt 当成父亲关系建出一棵树（即“灭绝树”），删掉 vvv 之后受影响的结点个数就是 vvv 的子树大小。所以现在只需要考虑 ttt 怎么求。

按拓扑序建树，则考虑到某个点的时候它的前驱已经考虑完了。那么它的父亲就会是它所有的前驱的 LCA （因为只有删掉公共祖先才能使这些前驱都被删掉，从而使 v 被删掉）。

边建树边倍增即可。复杂度 O(nlog⁡n)O(n\log n)O(nlogn) 。
