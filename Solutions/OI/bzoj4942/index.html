BZOJ4942 [NOI2017] 整数

发表于 2018-03-19 更新于 2022-06-24 分类于 Solutions 阅读次数：

Description

你要维护一个二进制数 xxx ，有 nnn 个操作：

  • 1 a b ，表示 xxx += a∗2ba * 2^ba∗2b ；
  • 2 k ，表示询问 xxx 第 kkk 个二进制位的值。

初始时 x=0x = 0x=0 。

n⩽106,∣a∣⩽109,b,k⩽3∗107n \leqslant 10^6, |a| \leqslant 10^9, b, k \leqslant 3*10^7n⩽106,∣a∣⩽109,b,k⩽3∗107

Solution

先考虑只有 a>0a > 0a>0 的时候怎么做。

考虑暴力：把 aaa 的每一位拆开加；每次要在第 iii 位上加的时候不停往前进位直到进不上去为止。

分析一下复杂度：

每次加的时候会把某一些二进制位从 1 变成 0 ，但是只有一个二进制位从 0 变成 1 。从 0 变成 1 的位至多有 O(nlog⁡a)O(n\log a)O(nloga) 个，它们一共也只会花 O(nlog⁡a)O(n\log a)O(nloga) 的时间变回 0 。所以总时间复杂度就是 O(nlog⁡a)O(n\log a)O(nloga) 的。

考虑到 bool 比较低效，我们可以压位 (323232 位压一个 unsigned int) ，那么 aaa 就可以拆成两个加上去。

再考虑如何处理减法。

如果我以上述方式维护 x=x0−x1x = x_0 - x_1x=x0​−x1​ ，其中 x0,x1x_0, x_1x0​,x1​ 分别是只考虑加和只考虑减（取绝对值）的值，那么查询时相当于查询 x0−x1x_0 - x_1x0​−x1​ 的第 kkk 位的值。

考虑 x0x_0x0​ 的后 kkk 位 (000 到 k−1k-1k−1 位) 与 x1x_1x1​ 的后 kkk 位的大小关系。如果前者比较小那么在第 kkk 位上需要退位；否则不需要。比较方式是找到后 kkk 位中的两个数不同的最高位比较。

那么我们只需要开一个 set 维护“两个数不同的位置”，那么“后 kkk 位中两个数不同的最高位”就相当于一个 lower_bound 。

时间复杂度 O(nlog⁡n)O(n\log n)O(nlogn) （因为至多改变 O(n)O(n)O(n) 位，所以 set 的大小也是 O(n)O(n)O(n) 的）。

空间复杂度 O(n+b)O(n + b)O(n+b) 。

目前在 LOJ 和 Luogu 榜首， BZOJ 第三。
