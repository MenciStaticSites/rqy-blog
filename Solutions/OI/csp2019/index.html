D1T1 格雷码

Description

给定 n,kn,kn,k，求 nnn 位二进制格雷码中第 kkk 个是多少。n≤64,0≤k<2nn\leq64,0\leq k<2^nn≤64,0≤k<2n。

Solution

可以递归求，也可以找规律得到答案为 k ^ (k >> 1)。

注意 kkk 需要用 unsigned long long 存储。

D1T2 括号树

Descrption

给出一棵有根树，每个结点上有一个括号。对每个点询问从根到它的括号字符串有多少个子串是合法括号串。

n≤105n\leq10^5n≤105

Solution

考虑计算根到每个点的字符串有多少个后缀是合法字符串，之后只需要求一个树上前缀和。

定义一个点的深度 dxd_xdx​ 为：从根到它的路径上左括号比右括号多多少个。

那么令 fx,if_{x,i}fx,i​ 表示从根到 xxx 的路径上有多少个点 yyy 满足：

  1. y…xy\dots xy…x 这一段路径上的括号串任意前缀的左括号个数都大于等于右括号个数。也就是说，这个串可以通过在右边添加一些右括号得到合法括号串；
  2. yyy 的深度等于 iii。

容易发现如果 xxx 上面为右括号，y…xy\dots xy…x 满足上述条件 1 且 dy=dx+1d_y=d_x+1dy​=dx​+1，则 y…xy\dots xy…x 是合法括号串。所以 xxx 的答案就是 fx,dx+1f_{x,d_x+1}fx,dx​+1​。

可以发现当 xxx 上面为左括号的时候

fx,i={ffax,ii≠dxffax,i+1i=dxf_{x,i}=\begin{cases}f_{fa_x,i}&i\neq d_x\\f_{fa_x,i}+1&i=d_x\end{cases} fx,i​={ffax​,i​ffax​,i​+1​i​=dx​i=dx​​

xxx 上面为右括号的时候

fx,i={ffax,ii≠dx0i=dxf_{x,i}=\begin{cases}f_{fa_x,i}&i\neq d_x\\0&i=d_x\end{cases} fx,i​={ffax​,i​0​i​=dx​i=dx​​

无论如何 fxf_{x}fx​ 和 ffaxf_{fa_x}ffax​​ 都只有一个位置不同，因此可以直接边 dfs 边修改 fff（回溯的时候改回去就可以了）。复杂度O(n)O(n)O(n)。

D2T1 Emiya 家今天的饭

Desciption

nnn 种烹饪方式 mmm 种食材，第 iii 种方式第 jjj 种食材有 ai,ja_{i,j}ai,j​ 种做法，要求做若干道（至少一道）菜使得每道菜的烹饪方式不同，且每种食材的菜不能超过一半（向下取整）。求共有多少种方式，对 998244353 取模。

n≤100,m≤2000,ai<998244353n\leq100,m\leq2000,a_i<998244353n≤100,m≤2000,ai​<998244353。

Solution

不考虑每种食材不超过一半的限制，答案是

∏i(1+∑jai,j)−1\prod_{i}\left(1+\sum_ja_{i,j}\right)-1 i∏​(1+j∑​ai,j​)−1

减去 1 是去掉一道菜都不做的方案。

显然只可能有一种菜超过一半，于是枚举这种菜，对每个方式做背包即可（记一维状态表示这种菜比别的菜多做了多少份）。

背包复杂度 O(n2)O(n^2)O(n2)，总复杂度 O(n2m)O(n^2m)O(n2m)。分治 FFT！

D2T2 划分

Description

给定一个长为 nnn 的正整数序列，要求把它划分成若干段分别求和后单调不降，最小化每一段的和的平方和。

n≤4×107n\leq4\times10^7n≤4×107。

Solution

结论：最优解同时一定也是所有合法解中最后一段的和最小的解。

这个结论我还不会证…虽然它看上去很正确…

有了这个结论之后可以令 SiS_iSi​ 表示 a1+⋯+aia_1+\dots+a_ia1​+⋯+ai​，fif_ifi​ 表示只靠虑 1…i1\dots i1…i 的时候最后一段最小可以是多少。那么

fi=min⁡{Si−Sj∣Si−Sj≥fj}f_i=\min\{S_i-S_j\mid S_i-S_j\geq f_j\} fi​=min{Si​−Sj​∣Si​−Sj​≥fj​}

显然会选择最大的合法的 jjj，并且条件相当于 Sj+fj≤SiS_j+f_j\leq S_iSj​+fj​≤Si​。单调队列即可。

不能直接记录答案，要 DP 之后沿着转移从 nnn 走回去求平方和（因为要高精度）。

复杂度 O(n)O(n)O(n)。

D2T3 树的重心

Description

给出一棵树，分割掉每条边之后在两边求重心，最后求所有重心的编号之和。如果有两个重心就都加上去。

（重心定义为以其为根、每个子树大小都不超过总大小的一半（下取整））

n≤3×105n\leq3\times10^5n≤3×105。

Solution

显然任意情况下两个重心一定是相邻的，我们先求出其中较靠下的那个重心，最后 check 一下其父亲是不是也是重心就可以了。

首先说明一个结论：任意一个树，以任意点为根，重心（所有重心）一定在根所在的重链上。证明省略，挺简单的。

我们选择一个重心作为树根。

接下来考虑每个点的子树的重心。令 fif_ifi​ 表示 iii 的字数的重心，wiw_iwi​ 表示 iii 的重儿子。可以发现 fif_ifi​ 一定是 fwif_{w_i}fwi​​ 的祖先，暴力跳就可以了（每条重链上都只会从下往上跳，所以总复杂度是 O(n)O(n)O(n)）。

复杂的是求每个点字数外的重心。下面用 rrr 表示树根（也是全树的重心）。

如果某个点 xxx 不在 wrw_rwr​ 的子树里，那么删掉 xxx 的子树之后 rrr 的重儿子一定还是 wrw_rwr​，从而重心还在原来那条重链上。把这条重链从上到下排列下来记作 p1(=r),p2(=wr),…,pkp_1(=r),p_2(=w_r),\dots,p_kp1​(=r),p2​(=wr​),…,pk​，那么删掉 xxx 的子树后，重心就是最靠下的点 pip_ipi​ 使得 sizpi≥⌈n−sizx2⌉siz_{p_i}\geq\lceil\frac{n-siz_x}{2}\rceilsizpi​​≥⌈2n−sizx​​⌉。可以直接预处理出对每个 ttt，最靠下的 sizpi≥tsiz_{p_i}\geq tsizpi​​≥t 的点是哪个，然后 O(1)O(1)O(1) 查询。

还剩下 wrw_rwr​ 的子树里的那些点。记 vrv_rvr​ 表示 rrr 的次大的儿子，我们“假装”把根节点的重链转而连到 vrv_rvr​ 上；如果 xxx 是 wrw_rwr​ 的子树里的点，那么有两种可能：

  1. 删掉 xxx 及其子树之后根节点的重儿子没变。但是这样的话既然本来重心就是 rrr，删掉 xxx 的子树显然不会使得重心反而往 xxx 的方向靠近。因此这时候重心还会是 rrr；
  2. 删掉 xxx 的子树后根节点的重儿子变了，那显然会变成 vrv_rvr​。于是重心会在 rrr，或者 vrv_rvr​ 的重链上。

总之我们可以假装把 rrr 的重儿子改成 vrv_rvr​，然后对重儿子子树里所有的点做一个像之前求不在 wrw_rwr​ 子树里的点的答案一样的做法。

所有处理都是线性的，总复杂度也仅仅是 O(n)O(n)O(n)。
